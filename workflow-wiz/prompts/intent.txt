1. **Primary Goal**: What is the main objective the user wants to achieve?

2. **Trigger Type**: How should this workflow be initiated?

   - "event": Triggered by an event (new email, form submission, etc.)
   - "manual": User manually triggers it

3. **Trigger Details**: Specific information about when/how to trigger (e.g., "every weekday at 9am", "when new email arrives")

4. **Data Sources**: Where does the input data come from? (Gmail, Sheets, Notion, webhooks, etc.)

5. **Data Destinations**: Where should the output/results go?

6. **Required Tools**: Which integrations are needed?
   Available tools: gmail, sheets, notion, webhook

7. **Key Actions**: What are the main steps or actions to perform?

8. **Complexity**: Assess the workflow complexity
   - "simple": 1-3 steps, single tool
   - "moderate": 4-6 steps, 2-3 tools, basic logic
   - "complex": 7+ steps, multiple tools, conditional logic, data transformations

IMPORTANT: Respond ONLY with valid JSON in this exact format:
{
  "goal": "Brief description of what user wants to achieve",
  "summary": "One sentence summary of the workflow",
  "trigger_type": "schedule|event|manual",
  "trigger_details": "Specific trigger information",
  "data_sources": ["source1", "source2"],
  "data_destinations": ["dest1", "dest2"],
  "required_tools": ["gmail", "sheets", "notion", "webhook"],
  "key_actions": ["action1", "action2", "action3"],
  "complexity": "simple|moderate|complex"
}

Do not include any text before or after the JSON.
